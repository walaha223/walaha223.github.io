SPEC.md — Site vitrine WalahaTracker Mali
1. Nom du projet

WalahaTracker — Site vitrine bêta

2. Objectif du site

Créer un site web de présentation pour la version bêta de WalahaTracker, une application mobile de suivi scolaire familial destinée au marché malien.

Le site doit présenter clairement l’application, expliquer son utilité, rassurer les écoles et les parents, puis inviter les utilisateurs à rejoindre la version bêta.

3. Positionnement

WalahaTracker est une application mobile malienne qui aide les parents, les écoles et les répétiteurs à mieux suivre la scolarité des enfants depuis un espace simple, organisé et accessible.

4. Langue

Le site est entièrement en français.

Aucune version anglaise n’est nécessaire pour la bêta.

5. Marché cible

Le site cible uniquement le Mali.

Il ne faut pas présenter WalahaTracker comme une solution internationale pour le moment.

6. Ce qu’il ne faut pas mentionner

Le site ne doit pas parler de :

WalahaTab ;
WalahaWatch ;
smartwatch ;
tablette éducative ;
fabrication matérielle ;
marché international ;
diaspora dans cette première version ;
autres produits Walaha.

Le site doit rester concentré sur :

WalahaTracker Application Mobile — Suivi scolaire familial au Mali.

7. Public cible
7.1 Parents

Parents maliens qui veulent suivre la scolarité de leurs enfants, consulter les bulletins, gérer les frais scolaires et communiquer avec l’école.

7.2 Écoles

Écoles privées, écoles communautaires, écoles fondamentales, lycées ou établissements scolaires qui veulent mieux organiser la communication avec les familles.

7.3 Répétiteurs

Répétiteurs qui veulent être visibles, proposer leurs services et accompagner les enfants dans le suivi scolaire.

Section 3 — Problème
Titre
Au Mali, le suivi scolaire est encore trop dispersé
Texte
Dans beaucoup de familles, les informations scolaires circulent encore par appels, papiers, messages WhatsApp ou échanges informels. Les parents ne reçoivent pas toujours les informations à temps, les écoles manquent d’outils simples pour communiquer avec les familles, et le suivi des enfants devient difficile.
Cartes problèmes

Créer 4 cartes :

Informations perdues
Les bulletins, messages et annonces ne sont pas toujours faciles à retrouver.
Communication difficile
Les échanges entre parents, école et répétiteurs sont souvent dispersés.
Suivi manuel des frais
Les frais scolaires et contributions sont parfois suivis sur papier ou par messages.
Manque de visibilité
Les parents ne savent pas toujours ce qui se passe réellement à l’école.
Section 4 — Solution
Titre
WalahaTracker centralise le suivi scolaire familial
Texte
WalahaTracker regroupe les informations importantes de chaque enfant dans une seule application : profil scolaire, école, classe, bulletins, frais, besoins, répétiteurs, messages et notifications.
Promesse
Moins de désordre. Plus de visibilité. Une meilleure collaboration autour de chaque enfant.

26. Google Analytics

Le site doit intégrer Google Analytics 4 afin de suivre le nombre de visiteurs, les pages vues, les sources de trafic et les actions importantes comme les clics sur les boutons de bêta. Google recommande d’ajouter le Google tag sur le site pour envoyer les données vers Google Analytics, et l’identifiant de mesure GA4 commence généralement par G-.

Objectifs de suivi

Le site doit permettre de suivre :

Nombre de visiteurs
Pages vues
Clics sur “Rejoindre la bêta”
Clics sur “Nous contacter sur WhatsApp”
Soumissions du formulaire de contact
Source du trafic : Google, Facebook, WhatsApp, lien direct, etc.
Ville / pays approximatif des visiteurs
Type d’appareil : mobile, tablette, ordinateur
Intégration technique

Ajouter le script Google Analytics dans le fichier :

index.html

Le script doit être placé dans la balise :

<head>

Google indique que le Google tag doit être copié dans le code de chaque page du site, immédiatement après l’élément <head> lorsqu’il est installé manuellement.

Code à prévoir
<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());

  gtag('config', 'G-XXXXXXXXXX');
</script>

Remplacer :

G-XXXXXXXXXX

par l’identifiant réel de WalahaTracker dans Google Analytics.

27. Événements Google Analytics personnalisés

Le site doit aussi suivre certains événements importants. Google Analytics permet d’envoyer des événements personnalisés avec la commande event de gtag.js.

Événements à suivre
click_beta_button
click_whatsapp_button
submit_contact_form
click_school_beta
click_parent_beta
click_repetiteur_beta
Exemple pour le bouton bêta
<a href="#contact" class="btn-primary" onclick="gtag('event', 'click_beta_button');">
  Rejoindre la bêta
</a>
Exemple pour WhatsApp
<a 
  href="https://wa.me/223XXXXXXXX" 
  class="btn-secondary"
  onclick="gtag('event', 'click_whatsapp_button');"
>
  Nous contacter sur WhatsApp
</a>
